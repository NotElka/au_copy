# WhatsApp-бот AU Copy — подробный запуск «по-настоящему»

Это развёрнутая инструкция к `ЧЕКЛИСТ.md` (часть 2). Здесь — по шагам, с
конкретными кликами, типичными ошибками и проверками после каждого этапа.

Цель: убрать три «временности» теста — 24-часовой токен, случайный адрес
туннеля и ограничение «только свои номера» — и запустить бота для всех клиентов.

> Порядок важен. Идём сверху вниз: сначала номер и токен (без них ничего не
> заработает), потом постоянный адрес, потом безопасность, в конце — Live.

---

## Шаг 0. Что нужно подготовить заранее

- [ ] **Реальный номер телефона** для бота. Требования:
  - НЕ привязан к обычному WhatsApp / WhatsApp Business на телефоне
    (если привязан — открой WhatsApp → Настройки → Удалить аккаунт, потом жди ~30 мин);
  - может **один раз** принять SMS или звонок с кодом подтверждения;
  - после подтверждения телефон можно выключить — номер живёт в облаке Meta.
- [ ] **Аккаунт Meta Business** (business.facebook.com) — заведётся автоматически
  при создании приложения, но проверь, что он есть.
- [ ] Документы компании (для Business Verification): название, сайт/соцсеть,
  подтверждение существования бизнеса (для ИП в РК обычно достаточно сайта и данных).

## Шаг 1. Подключить свой номер вместо тестового

1. developers.facebook.com → твоё приложение → **WhatsApp → API Setup**.
2. В блоке **From** нажми **Add phone number**.
3. Заполни профиль бизнеса:
   - **Business name** — название (например, `AU Copy`);
   - **Display name** — имя, которое увидят клиенты как отправителя (например, `AU Copy Print`).
     ⚠️ Display name Meta проверяет вручную (от минут до 1–2 дней). Не пиши «Test»,
     не используй чужие бренды — отклонят.
   - часовой пояс, категория.
4. Введи **сам номер телефона** → выбери способ **SMS** или **Звонок** → получи код → введи код.
5. После подтверждения номер появится в списке **From**. Рядом будет новый
   **Phone number ID** — это число тебе и нужно (НЕ сам номер).

**→ Записать новый `Phone number ID`.** Старый тестовый `1284728404715519` больше не используем.

**Проверка:** в списке From виден твой номер со статусом «Connected».

## Шаг 2. Постоянный токен (System User) вместо 24-часового

Временный токен из API Setup живёт 24 часа — для прода не годится (ты уже ловил
`401 / code 190 Authentication Error`, когда он протух). Делаем бессрочный.

1. business.facebook.com → **Business Settings** (Настройки компании).
2. Слева **Users → System Users** → **Add** → имя (например, `aucopy-bot`),
   роль **Admin** → создать.
3. Выбери созданного System User → **Add Assets**:
   - вкладка **Apps** → выбери своё приложение → включи **Full control / Manage**;
   - вкладка **WhatsApp Accounts** → выбери свой WhatsApp Business Account →
     включи полный доступ.
4. Нажми **Generate new token**:
   - выбери своё **приложение**;
   - срок действия — **Never** (бессрочно);
   - отметь права (permissions):
     - ✅ `whatsapp_business_messaging`
     - ✅ `whatsapp_business_management`
5. **Скопируй токен сразу** — Meta покажет его только один раз.

**→ Это и есть постоянный `WHATSAPP_ACCESS_TOKEN`.**

**Проверка токена** (PowerShell, подставь свои значения):
```powershell
$tok = "ВСТАВЬ_ТОКЕН"
$pid = "НОВЫЙ_PHONE_NUMBER_ID"
Invoke-RestMethod -Uri "https://graph.facebook.com/v21.0/$pid" `
  -Headers @{ Authorization = "Bearer $tok" }
```
Если вернулись данные номера (а не ошибка 190) — токен рабочий.

## Шаг 3. Прописать новые значения в проекте

Открой `whatsapp/secrets.env` и обнови:

```env
WHATSAPP_ACCESS_TOKEN=<постоянный токен из шага 2>
WHATSAPP_PHONE_NUMBER_ID=<новый Phone number ID из шага 1>
WHATSAPP_VERIFY_TOKEN=<любая своя строка, например aucopy-prod-7h3k>
WHATSAPP_APP_SECRET=<см. шаг 5>
WHATSAPP_API_VERSION=v21.0
BACKEND_URL=http://127.0.0.1:8000
UPLOAD_TOKEN=<своя секретная строка, см. шаг 5>
WHATSAPP_PORT=8001
```

⚠️ Файл `secrets.env` уже в `.gitignore` — не коммить его.

## Шаг 4. Постоянный адрес (вместо случайного trycloudflare)

Бесплатный `trycloudflare` даёт **новый адрес при каждом перезапуске** — каждый
раз пришлось бы заново прописывать вебхук в Meta. В проде так нельзя. Выбери один вариант:

### Вариант A — ngrok со статическим доменом (проще всего, есть 1 бесплатный)
1. Зарегистрируйся на ngrok.com → раздел **Domains** → создай бесплатный
   статический домен (вид `что-то.ngrok-free.app`).
2. Установи и авторизуй:
   ```powershell
   winget install ngrok.ngrok
   ngrok config add-authtoken <твой токен с сайта ngrok>
   ```
3. Запуск:
   ```powershell
   ngrok http 8001 --domain=что-то.ngrok-free.app
   ```
   Адрес теперь постоянный.

### Вариант B — cloudflared named tunnel (нужен свой домен в Cloudflare)
1. Привяжи домен к Cloudflare.
2. `cloudflared login` → `cloudflared tunnel create aucopy` →
   `cloudflared tunnel route dns aucopy bot.твойдомен.com`.
3. Настрой `config.yml` (service: `http://localhost:8001`) → `cloudflared tunnel run aucopy`.

### Вариант C — VPS/хостинг с белым IP и доменом (самый стабильный, для боевого салона)
Поднять бэкенд + бота на сервере, выдать домен с HTTPS (Caddy/Nginx + Let's Encrypt).

**→ Записать постоянный адрес, например `https://aucopy.ngrok-free.app`.**

## Шаг 5. Безопасность (обязательно для прода)

1. **App secret** (для проверки подписи вебхука):
   - developers.facebook.com → приложение → **App settings → Basic** →
     поле **App Secret** → Show → скопировать → вписать в `WHATSAPP_APP_SECRET`.
   - После этого бот начнёт проверять `X-Hub-Signature-256` — посторонний не
     сможет слать фейковые события на твой вебхук.
2. **UPLOAD_TOKEN** — поменяй с дефолтного `dev-secret-change-me` на свою строку
   в **двух местах одинаково**:
   - `backend/config.py` → `UPLOAD_TOKEN = "..."`
   - `whatsapp/secrets.env` → `UPLOAD_TOKEN=...`
   Если значения разойдутся — бэкенд вернёт `401` и файл не загрузится.
3. Проверь, что `secrets.env`, `backend/kaspi_token.json`, `cloud/cloud.db` —
   в `.gitignore` (уже так).

## Шаг 6. Запустить всё и подключить вебхук

Три процесса (три окна PowerShell, из корня проекта):

```powershell
# 1. Бэкенд
& .\venv\Scripts\python.exe -m backend.run

# 2. Бот
& .\venv\Scripts\python.exe -m whatsapp.run

# 3. Постоянный туннель (пример для ngrok)
ngrok http 8001 --domain=aucopy.ngrok-free.app
```

Подключить вебхук в Meta (делается один раз, адрес теперь не меняется):
1. **WhatsApp → Configuration → Webhook → Edit**:
   - **Callback URL** = `https://aucopy.ngrok-free.app/webhook`
   - **Verify token** = та же строка, что `WHATSAPP_VERIFY_TOKEN`
   - **Verify and save** → зелёная галочка.
2. Там же **Webhook fields → Subscribe** на поле **messages**.

**Проверка вебхука** (должен ответить):
```powershell
Invoke-RestMethod "https://aucopy.ngrok-free.app/health"
# → ok = True
```

## Шаг 7. Перевести приложение в Live + Business Verification

Пока приложение в режиме **Development**, действует allowed-list (бот пишет только
подтверждённым номерам). Чтобы писать **любым** клиентам:

1. **Business Verification**:
   - developers.facebook.com → приложение → **App Review → Business Verification**
     (или в Business Settings → **Security Center**).
   - Заполни данные компании, приложи подтверждение (сайт, реквизиты) → отправь.
   - Проверка Meta: от пары часов до нескольких дней.
2. **Перевести приложение в Live**:
   - вверху страницы приложения переключатель **App Mode: Development → Live**.
3. **Платёжный метод**: WhatsApp → привязать карту (Payment settings). Переписка
   сверх бесплатного лимита — платная (сервисные разговоры по тарифам Meta).

**После этого:** allowed-list больше не нужен — боту может писать любой номер.

**Проверка:** попроси знакомого с номером, которого НЕТ в allowed-list, прислать
боту PDF → должен прийти код. Если пришёл — прод работает.

## Шаг 8. Автозапуск (чтобы не поднимать руками)

Чтобы бэкенд, бот и туннель стартовали сами при включении ПК:
- **Планировщик заданий Windows** (Task Scheduler): три задачи «At startup»,
  каждая запускает свою команду; или
- обернуть в службы (например, через NSSM); или
- на VPS — `systemd`-юниты.

## Частые ошибки (по кодам)

| Ошибка | Что значит | Что делать |
|---|---|---|
| `401 / code 190 Authentication Error` | токен протух или неверный | сделать постоянный токен (шаг 2) |
| `(#131030) not in allowed list` | приложение в Development, номер не добавлен | добавить номер в allowed-list **или** перейти в Live (шаг 7) |
| `(#131047)` re-engagement | прошло >24 ч с сообщения клиента | ответить можно только шаблоном (template), а не свободным текстом |
| вебхук «Verify failed» | не совпал Verify token | строка в Meta = `WHATSAPP_VERIFY_TOKEN` в `secrets.env` |
| файл не грузится, бэкенд `401` | разошёлся `UPLOAD_TOKEN` | одинаковая строка в `backend/config.py` и `secrets.env` |

## Итоговая таблица «тест → прод»

| | Тест | Продакшен |
|---|---|---|
| Номер бота | даёт Meta | свой реальный, подтвердить 1 раз (шаг 1) |
| Токен | временный, 24 ч | постоянный System User (шаг 2) |
| Адрес | случайный trycloudflare | постоянный ngrok/named tunnel/VPS (шаг 4) |
