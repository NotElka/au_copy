# WhatsApp-бот AU Copy — что нужно

Краткая шпаргалка: отдельно «для теста» и отдельно «чтобы работало по-настоящему».
Подробные клики по каждому шагу — в `whatsapp/README.md`.

---

## 🧪 ЧАСТЬ 1. Для теста (бесплатно, за ~15 минут)

В тестовом режиме боту можно писать **только с твоих личных номеров**, которые
ты сам добавишь в Meta. Этого достаточно, чтобы убедиться, что всё работает.

### Что нужно иметь
- [ ] Аккаунт **Facebook** (обычный личный — для входа в developers.facebook.com).
- [ ] **Твой телефон с установленным WhatsApp** (на него бот будет отвечать).
      ⚠️ Отдельную симку/номер для бота покупать НЕ нужно — Meta даёт тестовый номер сама.
- [ ] Установленный **cloudflared** (туннель):
      `winget install --id Cloudflare.cloudflared`

### Что сделать в Meta
- [ ] Создать приложение: developers.facebook.com → My Apps → Create App → тип **Business**.
- [ ] Подключить продукт **WhatsApp** (карточка → Set up). Meta заведёт тестовый номер.
- [ ] В **WhatsApp → API Setup** скопировать:
  - [ ] **Temporary access token** (живёт 24 часа — для теста ок).
  - [ ] **Phone number ID** (число под тестовым номером, это НЕ сам номер).
- [ ] Там же в блоке «To» добавить **свой личный номер** и подтвердить его
      (Meta пришлёт код в WhatsApp).

### Что сделать в проекте
- [ ] Скопировать `whatsapp/secrets.example.env` → `whatsapp/secrets.env`.
- [ ] Вписать в `secrets.env`:
  - [ ] `WHATSAPP_ACCESS_TOKEN=` — токен из API Setup
  - [ ] `WHATSAPP_PHONE_NUMBER_ID=` — Phone number ID
  - [ ] `WHATSAPP_VERIFY_TOKEN=` — придумать любую строку (запомнить)
  - [ ] `UPLOAD_TOKEN=` — должен совпадать с `backend/config.py` (по умолчанию `dev-secret-change-me`)

### Запустить (3 окна PowerShell)
- [ ] Бэкенд:   `& .\venv\Scripts\python.exe -m backend.run`
- [ ] Бот:      `& .\venv\Scripts\python.exe -m whatsapp.run`
- [ ] Туннель:  `cloudflared tunnel --url http://localhost:8001`
      → скопировать выданный адрес `https://xxxx.trycloudflare.com`

### Подключить вебхук в Meta
- [ ] **WhatsApp → Configuration → Webhook → Edit**:
  - [ ] Callback URL = `https://xxxx.trycloudflare.com/webhook`
  - [ ] Verify token = та же строка, что `WHATSAPP_VERIFY_TOKEN`
  - [ ] Verify and save → должна появиться зелёная галочка
- [ ] Там же подписаться на поле **messages** (Subscribe).

### Проверка
- [ ] Написать боту на тестовый номер и прикрепить PDF/фото → в ответ придёт код.

> ⚠️ Минусы теста: токен протухает каждые **24 часа**, бесплатный туннель даёт
> **новый адрес при каждом перезапуске** (тогда вебхук в Meta надо прописать заново),
> и писать боту можно только со своих добавленных номеров.

## 🚀 ЧАСТЬ 2. Чтобы бот работал по-настоящему (для всех клиентов)

Здесь нужно убрать три «временности» из теста: 24-часовой токен,
случайный адрес туннеля и ограничение «только свои номера».

### Номер бота
- [ ] **Реальный номер телефона** для бота, который:
  - НЕ зарегистрирован в обычном WhatsApp / WhatsApp Business на телефоне;
  - может один раз принять код подтверждения (SMS или звонок).
  - После регистрации работает из облака — держать телефон включённым НЕ нужно.
- [ ] Подтвердить этот номер в **WhatsApp → API Setup** (Add phone number).

### Постоянный токен (вместо 24-часового)
- [ ] Meta → **Business Settings → System Users** → создать System User.
- [ ] Выдать ему доступ к приложению, сгенерировать токен с правами
      `whatsapp_business_messaging` и `whatsapp_business_management`.
- [ ] Вписать этот токен в `WHATSAPP_ACCESS_TOKEN` (он бессрочный).

### Постоянный адрес (вместо случайного trycloudflare)
Нужен стабильный HTTPS-адрес, который не меняется при перезапуске. Варианты:
- [ ] **cloudflared named tunnel** — нужен свой домен, привязанный к Cloudflare; или
- [ ] **ngrok со статическим доменом** (1 бесплатный):
      `ngrok http 8001 --domain твой.ngrok-free.app`; или
- [ ] полноценный хостинг/VPS с белым IP и доменом.

### Безопасность (обязательно для прода)
- [ ] Заполнить `WHATSAPP_APP_SECRET` (Meta → App settings → Basic → App secret) —
      бот начнёт проверять подпись вебхука `X-Hub-Signature-256` (защита от подделок).
- [ ] Сменить `UPLOAD_TOKEN` в `backend/config.py` и в `secrets.env` на что-то своё
      (не оставлять `dev-secret-change-me`).
- [ ] Не коммитить `secrets.env` (уже в `.gitignore`).

### Верификация бизнеса в Meta
- [ ] Пройти **Business Verification** (Meta проверит твою компанию) — без этого
      есть лимиты на количество сообщений и список получателей.
- [ ] После верификации боту смогут писать **любые** номера, не только добавленные.

### Автозапуск (чтобы не запускать руками)
- [ ] Сделать так, чтобы бэкенд, бот и туннель поднимались сами при старте ПК
      (например, через «Планировщик заданий» Windows или как службы).

## Короткий итог

| | Тест | Продакшен |
|---|---|---|
| Номер бота | даёт Meta (бесплатно) | свой реальный номер, подтвердить 1 раз |
| Токен | временный, 24 ч | постоянный (System User) |
| Адрес (туннель) | случайный trycloudflare | постоянный (named tunnel / ngrok / VPS) |
| Кому отвечает | только своим номерам | любым (после Business Verification) |
| Подпись вебхука | можно пропустить | обязательно `WHATSAPP_APP_SECRET` |
